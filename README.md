# matsim-schemas

This repository is the source of truth for MATSim schema definitions and
generated language bindings.

## Repository Layout

`proto/` contains the canonical Protocol Buffer definitions. It is also the
public protobuf import root, so imports from other repositories should be
resolved relative to `proto/`.

Language-specific packages are generated from those schemas:

- Java classes are generated by the Maven module in `java/`.
- Rust message types are generated by the Cargo crate in `rust/`.

Other repositories may still use `proto/` directly for their own code
generation when they need bindings for another language or a different build
system.

## Java Consumers

Once published, Java consumers should depend on:

```xml
<dependency>
  <groupId>org.matsim</groupId>
  <artifactId>matsim-schemas-java</artifactId>
  <version>...</version>
</dependency>
```

Consumers should pin released versions instead of depending on `main`.

## Rust Consumers

Once published, Rust consumers should depend on the `matsim-schemas` crate.
This crate generates plain protobuf message types with `prost`. It does not use
`tonic-build`, and it does not generate gRPC service clients or servers.

Consumers should pin released versions instead of depending on `main`.

## Downstream Rust Services With Tonic

Rust services that use `tonic` should define their own service `.proto` files
in their own repository. Those service files may import MATSim schema files for
`protoc` include resolution, but they should map MATSim protobuf packages to
this crate with `prost_build::Config::extern_path`. That avoids duplicate
generated Rust types.

Example downstream `build.rs`:

```rust
let mut config = prost_build::Config::new();

config.extern_path(
    ".matsim.simulation.io.types",
    "::matsim_schemas::matsim::simulation::io::types",
);

tonic_build::configure()
    .build_client(true)
    .build_server(true)
    .compile_protos_with_config(
        config,
        &["proto/my/service/v1/service.proto"],
        &[
            "proto",
            "../matsim-schemas/proto",
        ],
    )
    .unwrap();
```
